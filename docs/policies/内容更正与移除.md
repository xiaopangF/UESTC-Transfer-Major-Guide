# 内容更正与移除

普通事实错误、失效链接或分类问题，可通过仓库的“流程或内容纠错”Issue 报告，并提供页面路径与公开来源。

版权、隐私或考试安全相关问题，可使用“内容报告”Issue 只提交受影响页面的 URL 和问题类型。不要在公开 Issue 中复述敏感信息、题目内容或个人信息。需要提供敏感证据时，请使用维护者公开的私下联系渠道。

维护者处理时将：

1. 确认受影响的文件、版本与风险类型。
2. 必要时先从当前分支移除或隐藏内容。
3. 联系投稿者或权利人核对可公开范围。
4. 选择更正、补充来源、匿名化或删除。
5. 严重泄露时评估 Git 历史重写与 GitHub 支持流程。

由于 Fork、缓存和本地克隆的存在，仓库不能承诺内容从所有副本中完全消失。

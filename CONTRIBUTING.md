# 贡献指南

感谢你帮助补全资料。仓库接受 Issue 投稿和 Pull Request，两种方式的内容标准相同。

## 投稿前检查

提交前请确认：

1. 内容是本人原创，或已清楚说明公开来源与授权情况。
2. 内容不涉及仍在进行的考核、内部题库、泄露材料或未经许可的复制品。
3. 已移除姓名、学号、手机号、邮箱、QQ/微信、准考证号、人脸和可识别他人的细节。
4. 时间、专业名称和数据口径没有凭印象补全；不确定的地方明确写“不确定”。
5. 同意原创部分按仓库对应许可证发布。

公开 GitHub 投稿无法实现真正匿名。即使正文署名为“匿名贡献者”，Issue、Pull Request、提交作者和 Git 邮箱仍可能公开。请先阅读 [隐私说明](PRIVACY.md)，并考虑在 GitHub 中启用 noreply 邮箱。

## 方式一：使用 Issue 表单

进入 [新建 Issue](../../issues/new/choose)，选择最接近的表单：

- 经验投稿：面试、准备过程或完整转专业经历。
- 机考回忆投稿：对应考核完全结束后，由考生用自己的语言整理的回忆版。
- 申请数据新增或纠错：新增一条统计记录，或纠正现有记录。
- 流程或内容纠错：更新流程、链接、表述或索引。

维护者会把符合要求的 Issue 整理进对应目录。请不要在公开 Issue 中粘贴需要私下处理的敏感信息。

## 方式二：提交 Pull Request

1. Fork 本仓库并新建分支。
2. 根据内容选择 [经验模板](templates/面试与转专业经验模板.md)、[机考模板](templates/机考回忆模板.md) 或 [年度流程模板](templates/年度流程模板.md)。
3. 文件名建议使用 `年份-学院-专业-主题.md`，路径和文件名尽量使用稳定的 ASCII 单词或拼音。
4. 在相应索引页增加一行链接。
5. 数据变更请同时运行本地校验。
6. 填写 Pull Request 模板。一条 PR 尽量只处理一个年份、一组数据或一篇经验。

建议命令：

```bash
python scripts/validate_data.py
python -m unittest discover -s tests -v
```

## 内容放置位置

| 内容类型 | 位置 |
| --- | --- |
| 总体流程 | `docs/process/总体流程.md` |
| 年度流程 | `docs/process/by-year/` |
| 申请人数数据 | `docs/data/历年申请数据.csv` |
| 计算机机考资料 | `docs/exams/computer/<年份>/` |
| 面试与完整经验 | `docs/experiences/<年份>/` |
| 可复制模板 | `templates/` |

Git 不保存空目录。第一次加入某个年份时，可直接创建对应年份目录和 Markdown 文件。

## 数据贡献要求

`历年申请数据.csv` 的一行代表“某一年度、某一批次、某一目标学院与专业、某一统计口径”的记录。字段定义见 [数据说明](docs/data/历年申请数据说明.md)。

- 未知值必须留空，只有来源明确写为零时才填 `0`。
- 不手工填写录取率，避免分子和分母口径不一致。
- 历史记录保留当年公告使用的学院、专业和方向名称。
- 每行必须有稳定且唯一的 `record_id`、主要来源和核验日期。
- 官方数据应链接学校或学院公开页面；非官方数据必须标注证据等级。
- 若录取人数大于申请人数，校验脚本只会警告，因为公开材料的统计口径可能不同；投稿者必须在 `notes` 中解释。

## 经验贡献要求

- 背景信息尽量使用区间，例如“排名前 20%”，避免精确绩点、班级和过细时间线组合识别出本人。
- 可以选择不公开转专业结果。
- 区分亲身经历、转述和个人推测。
- 不评价或识别具体教师、工作人员与其他考生。
- 面试问题宜概括考察主题；若记录具体表述，请说明记忆可能有偏差。

## 审核标准

维护者主要检查格式、可追溯性、隐私、版权和内容边界。合并不代表对经验效果或事实完整性的保证。对存在争议的内容，优先保留来源与不同口径，而不是把单一说法写成确定事实。

## 许可与署名

提交即表示你有权发布该内容，并同意：

- 原创文档、经验、回忆题和数据整理按 CC BY 4.0 发布。
- `scripts/` 与 `.github/workflows/` 中的代码按 MIT License 发布。
- 可在模板中选择 GitHub 用户名、化名或“匿名贡献者”作为页面署名；Git 历史中的公开身份信息不受页面署名控制。
